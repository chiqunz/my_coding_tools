# Python Database Migration Skill

## When Triggered

Create or manage database migrations when schema changes are needed.

## Steps

1. **Create/modify the SQLAlchemy model** in `src/app/models/`:
   ```python
   from sqlalchemy import String, Integer, DateTime
   from sqlalchemy.orm import Mapped, mapped_column
   from app.models.base import Base

   class Item(Base):
       __tablename__ = "items"

       id: Mapped[int] = mapped_column(primary_key=True)
       name: Mapped[str] = mapped_column(String(255))
       description: Mapped[str | None] = mapped_column(String(1000), nullable=True)
       created_at: Mapped[datetime] = mapped_column(DateTime, server_default=func.now())
   ```

2. **Generate the migration**:
   ```bash
   uv run alembic revision --autogenerate -m "add_items_table"
   ```

3. **Review the generated migration** file in `alembic/versions/`:
   - Verify the `upgrade()` and `downgrade()` functions
   - Ensure indexes are created for frequently queried columns
   - Check foreign key constraints are correct

4. **Apply the migration**:
   ```bash
   uv run alembic upgrade head
   ```

5. **Verify**:
   ```bash
   uv run alembic current  # Check current revision
   uv run alembic history   # View migration history
   ```

## Key Conventions

- Use SQLAlchemy 2.0+ `Mapped[]` type annotations (not legacy Column())
- Always include `downgrade()` for reversibility
- Migration messages should be descriptive: `"add_user_email_index"` not `"update"`
- Never modify an already-applied migration — create a new one
- Use `server_default` for database-level defaults, not Python-level defaults
- Always review autogenerated migrations before applying
